# Decode Patterns — JD 话术翻译表 + Hidden Signal 词典

两个用途：① 在 Decoder 流程把营销话术翻译成真实需求；② 从 JD 措辞推断招聘经理找的"人"。

---

## 一 · 营销话术 → 真实需求 翻译表

招聘 JD 几乎从不直白说话。下面是高频话术的真实含义：

| JD 写 | 真实含义 / 解读 |
|---|---|
| "strong experience with X" | 做过 X 1-2 次能讲清楚即可，不需要 X 年专家 |
| "deep expertise in X" | 真要 5+ 年 X 经验，且能 reference check 出来 |
| "passion for X" | 没硬要求经验，但你得能讲出对 X 的真诚兴趣 |
| "working knowledge of X" | 听说过 + 能聊，不要求做过 |
| "exposure to X" | 完全可有可无的 nice to have |
| "you've shipped …" 列了具体产物 | 必须有可秀的作品集 / case study |
| "thrive in fast-paced environment" | 节奏快 / 经常变方向 / 可能加班 |
| "comfortable with ambiguity" | 没人给你需求文档 / 自己定义问题 |
| "wear many hats" | 人手不够 / 边界模糊 / 早期阶段 |
| "self-starter / self-directed" | 没有 micromanage / 你得自己 push |
| "collaborate cross-functionally" | 政治 / 协调成本高 / 沟通工作量大 |
| "data-driven" | 决策得用数据 backing，不能只靠直觉 |
| "user-obsessed / customer-obsessed" | 决策时用户优先于内部 stakeholder |
| "high bar / craft / pixel-perfect" | 评审严格 / 多轮迭代 / 作品集会被细抠 |
| "scale" / "scaled X to Y" | 需要做过大规模产品 / 大用户量 |
| "0 to 1" | 没有现成 framework / 探索期 |
| "1 to N" / "scaling stage" | 有产品 PMF，要做增长和成熟化 |
| "transform" / "modernize" | 旧系统 / legacy / 改造期 |
| "thought leadership" | 期望对外发声 / 写作 / 演讲 |
| "mentor / coach others" | 至少要 senior 以上 / 带过人 |
| "player-coach" | 既要自己做 IC 工作又要带人，工作量 1.5x |

## 二 · Hidden Signal 词典（最值钱）

JD 反复出现这些词 / 句式时，能推断招聘经理的"人物画像偏好"。

### 自驱型 vs 执行型

| 词 / 句式 | 推断 |
|---|---|
| ambiguity / unstructured / no playbook / undefined | **找自驱型**，能在没明确需求时自己定义问题 |
| ownership / accountable / drive / end-to-end | **要全程负责**，不能只做 piece |
| comfortable wearing many hats | **角色边界模糊**，可能要做 JD 之外的事 |
| structured / process / framework / established | **找执行型**，有 process，按 process 走 |

### 协作密度

| 词 / 句式 | 推断 |
|---|---|
| influence / align / partner / persuade | **政治密度高**，跨团队协作和说服很重要 |
| stakeholder management / cross-functional | **报告关系复杂**，对上对下对外都得搞定 |
| autonomous / independent work | **个人贡献为主**，少 meeting 文化 |
| highly collaborative team | **会议多** / 集体决策 / 共识文化 |

### 节奏与压力

| 词 / 句式 | 推断 |
|---|---|
| fast-paced / move quickly / ship fast / scrappy | **节奏快**，可能加班 / 早期阶段 |
| iterate quickly / fail fast | **试错文化**，允许犯错但要快速调整 |
| high bar / high standards / craft / quality | **高要求**，多轮评审 / 反复打磨 |
| thoughtful / deliberate / measured | **节奏慢** / 重决策质量 |

### 文化倾向

| 词 / 句式 | 推断 |
|---|---|
| mission-driven / impact / change the world | **使命感重**，可能薪资略低 / 文化匹配会被问 |
| inclusive / diverse / belonging | **DEI 文化重**，可能被问相关问题 |
| ego-less / humble / no-asshole | **过去出过协作问题**，会重点考察软技能 |
| bold / opinionated / strong POV | **要敢于推动观点**，没观点的人会被淘汰 |
| customer-obsessed / user first | **决策时用户压倒一切** |
| metrics-driven / data-informed | **决策要 backing**，"我觉得"不够 |

### 团队阶段信号

| 词 / 句式 | 推断 |
|---|---|
| build the team / founding member / first X hire | **早期 / 0-1**，需要自己 set up 一切 |
| scale / grow / expand the team | **扩张期 / 1-N**，有 playbook 但要复制 |
| mature / established / optimize | **成熟期**，重维护和优化 |
| transform / rebuild / modernize | **转型期**，旧系统 + 改造工作量大 |

### Level / Title 信号（暗示真实 level）

| JD 暗示 | 实际 Level 推断 |
|---|---|
| "manage other designers" / "team of N" | Manager / Lead 以上 |
| "set vision / strategy" | Principal / Director |
| "execute against roadmap" | IC，可能 Mid–Senior |
| "ship features" 没提 vision | IC Mid 居多 |
| "thought partner to CEO/CPO" | Principal / Lead |
| "first X hire" | 看似 IC 但实际要做 Lead 的活 |

## 三 · 反向解读（红旗信号）

JD 里这些表达要警惕——可能在掩盖问题：

| JD 写 | 警惕什么 |
|---|---|
| "wear many hats" + 列了 5 个职责 | 一个人当三个人用 |
| "fast-paced" + "no work-life balance trade-off" | 加班严重 |
| "we're like a family" | 边界模糊 / 加班文化 |
| 反复出现 "rockstar / ninja / 10x" | 不专业 / 文化幼稚 |
| 薪资范围跨度极大（如 $80k–$250k） | level 含糊 / lowball 风险 |
| JD 写了 N 个 must have（10+ 条） | HM 不知道自己要什么，简历筛得很随机 |
| 招了 6 个月以上还在招 | 内部对 JD / level / 薪资有分歧，面试容易反复 |
| "looking for someone who can do X + Y + Z" 跨度极大 | 想招一个人省钱，进去后职责会撕扯 |

## 四 · 国内 JD 词典补充（中文招聘场景）

| 国内 JD 写 | 真实含义 |
|---|---|
| 拥抱变化 | 需求经常变 / 老板想法多 |
| 结果导向 | 不看过程看结果 / 加班是常态 |
| 抗压能力强 | 压力大 / 节奏紧 |
| 狼性 / 战斗力强 | 加班 / 强 OKR / 高流动 |
| 主人翁意识 | 期望 ownership，但通常给不了对应权限 |
| 复合型人才 | 一个人当多个人用 |
| 扁平化管理 | 可能是真扁平，也可能是"老板直接管 30 个人" |
| 高速发展期 | 早期 / 业务模式还在变 |
| 0-1 业务 | 没现成 framework，定义工作量大 |
| 平台型业务 | 成熟 / 螺丝钉风险 |
