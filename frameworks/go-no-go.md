# Go / No-Go — Should I Apply 打分法 + 拿面概率估算

两件事：① 推荐指数 ⭐ 怎么打；② 预估拿面概率怎么估。**目标不是精确预测，是帮用户做决策。**

---

## 一 · 推荐指数 ⭐ 1-5 打分

推荐指数 = 综合 5 个维度，**每个维度 0-1 分**，加权求和后映射到 ⭐。

### 维度 1 · Match Quality（权重 30%）

基于匹配 的 Match Score：

| Match Score | 分数 |
|---|---|
| 85%+ | 1.0 |
| 70-85% | 0.8 |
| 55-70% | 0.5 |
| 40-55% | 0.3 |
| < 40% | 0.0 |

### 维度 2 · Career Trajectory Fit（权重 25%）

这个职位对用户**长期路径**的价值：

- 是不是用户想去的方向？
- 是不是上升 / 持平 / 下降 level？
- 进去能学到什么 / 认识什么人 / 拿到什么 brand？

打分：
- 1.0 = 显著推进长期目标（方向对 + level 上升 + 学习曲线陡）
- 0.7 = 横向移动但加分（同 level 但更好的公司 / 团队 / 方向）
- 0.4 = 中性（换工作而已）
- 0.0 = 偏离长期路径或下降

### 维度 3 · Risk / Downside（权重 20%）

下行风险有多大：

- 公司财务 / 业务稳定性
- 团队动荡（lead 换人 / 重组 / 裁员）
- 业务方向不确定
- 入职后真实工作可能与 JD 差距大
- Title / Level inflation 风险

打分（反向）：
- 1.0 = 几乎无下行风险
- 0.7 = 一些风险但可控
- 0.4 = 有明显风险（需要尽调）
- 0.0 = 红旗多（建议放弃）

### 维度 4 · Compensation / Terms（权重 15%）

薪资 / 股权 / 福利 / 地点是否符合用户硬约束：

- 1.0 = 超过用户期望
- 0.7 = 符合期望
- 0.4 = 略低但其他因素可以弥补
- 0.0 = 明显低于底线（除非有其他强动机）

### 维度 5 · Opportunity Cost（权重 10%）

机会成本 / 时机：

- 这类机会稀缺度（一年能遇到几次）
- 用户当前 pipeline 状况（手里没机会 vs 手里 3 个 offer）
- 时间紧迫度（裁员中 vs 慢慢看）

打分：
- 1.0 = 稀缺机会 + 用户急
- 0.7 = 不算稀缺但值得花精力
- 0.4 = 一般
- 0.0 = 用户有更好的 / 这种机会很多

### 映射到 ⭐

```
综合分 = 0.30×Match + 0.25×Trajectory + 0.20×(1-Risk) + 0.15×Comp + 0.10×OppCost

⭐⭐⭐⭐⭐ (5/5)  → 综合分 ≥ 0.85    — 罕见，几乎无 downside
⭐⭐⭐⭐☆ (4/5)   → 0.70-0.85       — 值得投，可能要先 tailor / 内推
⭐⭐⭐☆☆ (3/5)   → 0.55-0.70       — 投也可不投也可，看用户其他选择
⭐⭐☆☆☆ (2/5)   → 0.40-0.55       — 不建议优先投，除非时间充裕
⭐☆☆☆☆ (1/5)    → < 0.40          — 不建议投
```

**5 星要慎给**——5 星意味着用户应该把这份当头号优先级。

## 二 · 拿面概率估算

**绝对原则：给区间，不给单点。** 拿面是高方差事件。

### 基线（按 Match Score 区间）

裸投、简历未 tailor 的基线：

| Match Score | 裸投拿面基线 |
|---|---|
| 85%+ | 30-45% |
| 70-85% | 15-30% |
| 55-70% | 8-15% |
| 40-55% | 3-8% |
| < 40% | < 3% |

> 基线已经很低——这反映行业现实。85% 匹配裸投还是有一半以上不会拿面（被淹没在简历海 / 内推优先 / 等等）。

### 乘数 / 调整因子

在基线上调整：

| 因素 | 调整 |
|---|---|
| 有内推（强内推：HM 直接看的） | ×2.5 ~ 3 |
| 有内推（普通内推：HR 备注） | ×1.5 ~ 2 |
| 简历 tailor 过 | ×1.3 ~ 1.5 |
| 公司当前招聘热度高（多个相关 JD / 公开扩张） | ×1.2 |
| 公司当前缩招 / hiring freeze 风险 | ×0.5 ~ 0.7 |
| 职位已挂超过 90 天 | ×0.7（HM 标准飘忽 / 内部混乱） |
| 用户有 deal-breaker（visa / 地理 / level mismatch） | ×0.3 ~ 0.5 |
| 用户 LinkedIn / portfolio 有强公开作品 | ×1.2 |
| 用户从 JD 路径过来很常见（公司是热门目标） | ×0.8（竞争激烈） |

### 输出格式

```
预估拿面概率：30-45%

基线计算（Match 78% → 15-30% 基线）
× 没有内推 → 不调整
× 简历会 tailor → ×1.4
× 公司刚拿融资在扩张 → ×1.2
× 职位挂出 2 周内 → 不调整
────────────────
区间 → 25-50% → 取保守区间 30-45%

如果能找到内推，可以拉到 50-70%。
```

**永远给"如何提升概率"的具体动作**——这是用户真正想要的。

## 三 · Deal-Breaker 清单

满足任一条 → 直接给 ⭐ ≤ 2，拿面概率打到极低：

- 用户 visa 状态不匹配 + 公司不 sponsor（JD 明写或公司历史）
- 职位地点用户硬性无法接受 + 不接受 remote
- 必须门槛（如必须有某 PE license / 必须 US citizen for clearance）缺失
- 薪资范围明显低于用户底线（且不在可谈范围）
- 用户与 JD 方向完全不沾边（< 30% Match）
- 用户曾被该公司 reject 过同岗位（< 12 个月内）

## 四 · 易错点

### 错误 1：把推荐指数等同于匹配度

匹配度高 ≠ 值得投。匹配度高但薪资低 / 方向错 / 公司在裁员，依然不值得投。

### 错误 2：把拿面概率说得太高

> "你简直完美匹配，95% 能拿面"——这种话过两天用户没拿面会怀疑整个 skill 的可信度。**永远保守给区间。**

### 错误 3：不敢给负面建议

用户来问"该不该投"——如果不该，**就老实说不该**。粉饰让用户浪费时间投到不该投的职位上。

### 错误 4：拍脑袋打分

每个分数都要能说出"依据是 JD 里的 ___ / Match Score 算出来的 ___"。说不出依据的分数 = 没分数。

### 错误 5：忽略机会成本

用户手里有 3 个 5 星机会 vs 0 个机会 vs 即将失业，同一份 JD 的"该不该投"答案完全不一样。要问清楚 pipeline 状况。
